Dory can be self-hosted on any machine that runs Docker. You own the data, choose the database backend, and configure every detail of the deployment — no SaaS account or external dependency required.

## Why self-host

- **Data privacy** — connection credentials, query history, and schema metadata never leave your infrastructure.
- **Control** — pin a version, set your own resource limits, and configure auth and encryption to meet your security requirements.
- **No SaaS limits** — no seat limits, usage caps, or subscription tiers imposed by a hosted service.

## Deployment modes

Dory ships as a single Next.js application with two runtime forms controlled by the `DORY_RUNTIME` environment variable.

## System requirements

For Docker deployment:

| Requirement | Minimum |
|-------------|----------|
| Docker | 20.10 or later |
| CPU | 1 vCPU |
| Memory | 512 MB RAM |
| Disk | 1 GB (plus space for PGLite data) |
| Network | Outbound access to your databases and AI provider API |

<Note>
  Dory does not require inbound internet access. All AI provider calls and database connections originate from the container outward.
</Note>

## Quick start

The fastest way to get Dory running is a single `docker run` command:

```bash
docker run -d --name dory \
  -p 3000:3000 \
  -e DS_SECRET_KEY="$(openssl rand -base64 32 | tr -d '\n')" \
  -e BETTER_AUTH_SECRET="$(openssl rand -hex 32)" \
  -e BETTER_AUTH_URL="http://localhost:3000" \
  -e DORY_AI_PROVIDER=openai \
  -e DORY_AI_MODEL=gpt-4o-mini \
  -e DORY_AI_API_KEY=your_api_key_here \
  -e DORY_AI_URL=https://api.openai.com/v1 \
  dorylab/dory:latest
```

Open `http://localhost:3000` once the container starts. See [Docker deployment](/self-hosting/docker) for the full setup guide including Docker Compose, volume mounting, and production configuration.

## Architecture

```
┌─────────────────────────────────────┐
│          Docker container           │
│                                     │
│   Next.js app (port 3000)           │
│   ├── bootstrap.mjs (migrations)    │
│   └── server.js                     │
│                                     │
│   Internal storage                  │
│   ├── PGLite (default, embedded)    │
│   └── PostgreSQL (optional, ext.)   │
└────────────────┬────────────────────┘
                 │ outbound connections
       ┌─────────┼──────────┐
       ▼         ▼          ▼
  ClickHouse  PostgreSQL  AI API
  MySQL       (your DBs)  (OpenAI etc.)
```

- **Next.js app** — the full Dory UI and API, compiled to a standalone Next.js output.
- **Bootstrap script** — runs database migrations on startup before the server starts.
- **Internal storage** — stores users, sessions, saved queries, and connection metadata. Uses embedded PGLite by default; can be pointed at an external PostgreSQL instance.
- **Your databases** — Dory connects outward to ClickHouse, PostgreSQL, and other databases you register as connections. These are never stored inside the container.
